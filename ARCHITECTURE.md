# Eddy_c — архитектура и спецификация (для доработки / переписывания на Verilog)

> Источник истины — реальные файлы проекта: `Eddy_c.bdf` (топ-схема), `mx_ctrl.tdf`,
> `*.tdf`/`*.vhd` модули, `Eddy_c.qsf` (пины) и документ «Описание форматов регистров и
> данных для платы вихретокового контроля».
> **Внимание:** таблица команд/SPI в `CLAUDE.md` — это *планируемая* переделка и НЕ
> совпадает с реальными опкодами. Ориентироваться на этот файл и на `mx_ctrl.tdf`.

Назначение платы: **вихретоковый контроль** — генерация формы воздействия через ЦАП и
синхронный захват отклика 4 каналами АЦП, обмен с ARM-контроллером (STM32F7) по SPI.

Устройство: Altera **Cyclone IV E EP4CE115F23C8**, Quartus II 11.0.

## 1. Интерфейс с ARM (STM32F7)

Обмен — 16-битными словами по **SPI** (STM32 SPI1), плюс параллельные регистры на порту PD.

| Сигнал | Пин FPGA | STM32 AF | Назначение |
|--------|----------|----------|------------|
| SCK    | PA5 *(см. прим.)* | SPI1_SCK  | такт SPI |
| SS     | PA4 | SPI1_NSS  | выбор кристалла (старт/конец слова) |
| MOSI   | PB5 | SPI1_MOSI | данные ARM→FPGA |
| MISO   | PB4 | SPI1_MISO | данные FPGA→ARM |
| PD[10:0] | Y8…AB7 | GPIO | параллельные регистры команды/частоты/канала |
| PB2 | — | GPIO | флаг «измерение завершено» (готовность по команде 2) |
| PE2, PE6 | — | GPIO | доп. управление |
| BRD[3:0] | AA11/AA12/AB11/AB12 | — | идентификация платы |

> Примечание: точное соответствие SCK/SS/MOSI/MISO ↔ PA4/PA5/PB4/PB5 нужно
> подтвердить по геометрии `Eddy_c.bdf` перед переписыванием (распиновка соответствует
> штатной SPI1, но направление и инверсии следует проверить).

### Параллельные регистры на порту PD (запись из ARM)
- **Регистр команд** — `PD[3:0]` (`cmd[3:0]`), 4 бита.
- **Регистр частоты дискретизации АЦП** — `PD[9:8]`, 2 бита:
  - `00` = 64 МГц, `01` = 32 МГц, `10` = 16 МГц, `11` = 8 МГц.
- **Регистр номера канала** — `PD[5:4]`, 2 бита (выбор канала для чтения АЦП).

### SPI-тракт (реализация)
- Приём (MOSI): `lpm_shiftreg16` — 16-бит сдвиговый влево, serial-in → `q[15:0]`.
- Передача (MISO): `lpm_shiftreg_out` — 16-бит, параллельная загрузка (`load`=`s_load`)
  результата `mux16_8.result[15:0]`, serial-out. Также участвует регистр типа `74165`.
- Внутренние стробы: `s_load`, `s_read`, `write_d`/`write_di`, `read_d`/`read_dd`.
- ⚠️ **К проверке для rewrite:** счётчик 16 тактов / детект границы слова (по SS или
  счётчику SCK) в `.bdf` явно не помечен — в новом коде реализовать аккуратный SPI-slave
  FSM (по CPOL/CPHA STM32 SPI1, MSB first).

## 2. Полная таблица команд (`cmd[3:0]`)

Запись/чтение «полезных данных» идёт по SPI; команда задаётся в `PD[3:0]`.
Чтение (`mux16_8.sel = mx_ctrl.mux_ctrl[2:0]`) подтверждено `mx_ctrl.tdf`.

| CMD | Тип | Действие |
|-----|-----|----------|
| 0  | NOP/read | Пустая команда; завершает предыдущую. (`mx_ctrl`: чтение режима, mux=000) |
| 1  | write | Запись формы в **ОЗУ ЦАП** с автоинкрементом адреса. Коды 0…16383 (14 бит). Запись `cmd=1` сбрасывает счётчик адреса ОЗУ. Запись — по SPI. |
| 2  | start | **Старт измерения**: пишет 16384 отсчёта в ОЗУ АЦП с автостопом по заполнению; одновременно проигрывает ОЗУ ЦАП (выдаёт воздействие). Длительность зависит от частоты дискретизации. Флаг окончания — `PB2`. |
| 3  | read | Чтение результатов из **ОЗУ АЦП**; канал — из регистра номера канала. (`mx_ctrl` mux=011) |
| 4  | read | Чтение **статусного слова** (см. §5). (`mx_ctrl` mux=110) |
| 5  | read | Чтение адреса сброса счётчика (проверка команды 14). (`mx_ctrl` mux=010) |
| 6  | ctrl | **Старт непрерывного** сигнала ЦАП (`d_run`=1). |
| 7  | ctrl | **Стоп непрерывного** сигнала ЦАП (`d_run`=0). |
| 8  | read | Чтение кодов двух задержек. (`mx_ctrl` mux=100) |
| 9  | write | Запись **регистра режима** `mode`. `mode[1:0]` = выбор каждого 1/4/16/64-го периода в режиме «4 точки»; `mode[7]` = «4 точки»/полная форма (см. §4). |
| 10 | read | Чтение кода **порога** компаратора ЦАП. (`mx_ctrl` mux=101) |
| 11 | write | Запись **регистра задержки тактов АЦП** (16 бит: младший байт = задержка0, старший = задержка1). |
| 12 | write | Запись **регистра порога** компаратора ЦАП (13 бит, рекоменд. 8192). |
| 13 | read | Чтение **ОЗУ ЦАП** с автоинкрементом адреса. (`mx_ctrl` mux=001) |
| 14 | write | Запись значения **сброса/предела счётчика адреса ЦАП** (14 бит, такт ЦАП 64 МГц). Одиночный режим: счётчик считает до этого значения → сброс в 0 и стоп. Непрерывный: счёт не останавливается. |
| 15 | reset | **Сброс адреса и содержимого ОЗУ АЦП**, по фронту установки команды. |

`cmd[3:0]` разворачивается декодером **`dc16`** (4→16) в линии `cmd0..cmd15`, которые
гейтируют действия записи/управления.

## 3. Карта чтения (мультиплексор `mux16_8`, sel = `mux_ctrl[2:0]`)

`mx_ctrl.tdf` переводит `cmd[3:0]` → `mux_ctrl[2:0]`:

| cmd | mux_ctrl | data#x читаемое слово |
|-----|----------|------------------------|
| 0  | 000 | `mode[15:0]` (регистр режима) — `data0x` |
| 13 | 001 | данные ЦАП `{gn,gn,DD[13:0]}` (`C1DD`, защёлка ОЗУ ЦАП) — `data1x` |
| 5  | 010 | `cmp[15:0]` = **значение предела адреса (регистр cmd14)**, НЕ живой счётчик — `data2x` |
| 3  | 011 | данные АЦП `{gn,gn,ADC[13:0]}` (канал из `PD[5:4]` через `mux13_4`) — `data3x` |
| 8  | 100 | `delay[15:0]` (две задержки) — `data4x` |
| 10 | 101 | `cmpr[15:0]` = порог компаратора `adc_cmp_h` (cmd12) — `data5x` |
| 4  | 110 | статусное слово (§5) — `data6x` |
| —  | 111 | `data7x` (не задействован) |

Выход `mux16_8.result[15:0]` (`O_DAT`) → `lpm_shiftreg_out`/`74165` → MISO.
Селектор мультиплексора — `mx_ctrl.mux_ctrl[2:0]` (= `MX_CTRL[2:0]`).

## 4. Тракт ЦАП и режимы «4 точки» / «полная форма»

**ОЗУ ЦАП = `DAC_RAM`** — `altsyncram` SINGLE_PORT, 16384×14, инициализация `DAC1.mif`
(адресуемая память, НЕ FIFO).

```
count_dac (14-бит, clk=64МГц, clk_en, aclr)
   q[13:0] ─► DAC_RAM.address
             DAC_RAM.data  ◄─ SPI (команда 1, запись формы)
             DAC_RAM.wren  ◄─ Write_DAC (cmd1 & строб записи)
             DAC_RAM.q[13:0] ─► DD[13:0] ─► внешний ЦАП
   q[13:0] ─► compar_addr.dataa ;  datab = регистр предела (cmd14)
             compar_addr.aeb (q==предел) ─► сброс адреса (+стоп в одиночном режиме)
```

- **Одиночный режим** (по команде 2 / по умолчанию): счётчик идёт до предела (cmd14),
  затем `aeb` сбрасывает его в 0 и останавливает.
- **Непрерывный режим** (`d_run`=1, команды 6/7): счётчик не останавливается — форма
  гонится по кругу.
- `freq[1:0]` (PD[9:8]) задаёт частоту проигрывания/дискретизации (делители `count256`,
  `cnt8`, `t_count`, `count3`).
- Конфиг 4 микросхем ЦАП — отдельный последовательный интерфейс `DAC_CLK`/`DAC_DATA`/
  `DAC_CS[3:0]` (не путать с шиной данных `DD`).

### «4 точки» vs «полная форма» — это `mode[7]` (команда 9)
- **`mode[7] = 0` → «4 точки»**: АЦП тактируется **стробом от компаратора ЦАП**
  (`compar_addr`/порог). Захват происходит только в выбранных точках формы воздействия →
  малое число отсчётов («точки»).
- **`mode[7] = 1` → «полная форма»**: АЦП пишет на **полной частоте дискретизации**
  (`freq[1:0]`), захватывается весь отклик.
- Выбор источника такта АЦП — мультиплексор (`mux21`/`mux4_1`), селектор = `mode[7]`.
- `mode[1:0]` в режиме «4 точки» задаёт периодичность выбора периода формы:
  `0` — каждый период, `1` — каждый 4-й, `2` — каждый 16-й,
  `3` — каждый 64-й. Эти биты не делят такт АЦП; они прореживают события
  пересечения порога до запуска двух задержек `delay0`/`delay1`.
- В компилируемом `Eddy_c.v` для режимов `1/2/3` выбраны разряды счётчика
  пересечений `[1]/[3]/[5]`, как в исходной BDF-схеме: интервалы 4/16/64.
- `delay0/delay1` (cmd11) — задержки строба АЦП относительно ЦАП (синхронизация момента
  выборки); через формирователи импульса (`imp_former1`).

> ⚠️ **К проверке для rewrite:** точная топология мультиплексора такта АЦП (per-channel
> или общий), и как именно `delay0/delay1` сдвигают строб — подтвердить по `.bdf`.

## 5. Статусное слово (команда 4), 16 бит

```
[15:8] = 0xA5            (идент; источник constAA55 = 0xAA55, старший байт 0xA5)
[6:5]  = freq[1:0]       код частоты дискретизации АЦП
[4]    = d_run           флаг непрерывного запуска ЦАП
[3:0]  = FULL3..FULL0    флаги заполнения ОЗУ АЦП по каналам
```
Дополнительно: `PB4` дублирует флаг канала 0; `PB2` — флаг окончания измерения (cmd2).
Флаги `EMPTY0..3` — пустота FIFO каналов. `gn` — бит усиления (§7).

## 6. Тракт АЦП (4 канала)

**ОЗУ АЦП = `ADC_RAM`** — `scfifo` (FIFO!), 16384×14, по одному на канал
(`full/empty/usedw`). Это принципиально иная память, чем адресуемое `DAC_RAM`.

```
C0D..C3D[13:0] + строб CxDCO (LVDS такты ADCx_CLKP/N на сами АЦП)
   каждый канал ─► свой ADC_RAM (FIFO):
        wrreq  ◄─ строб такта АЦП  &  «идёт измерение» (старт cmd2)
        rdreq  ◄─ чтение по SPI (cmd3 + номер канала)
        aclr   ◄─ cmd15 (сброс содержимого)
        full   ─► FULLk (в статус; авто-стоп записи при заполнении)
   чтение: mux13_4 (sel = PD[5:4]) ─► {gn,gn,ADC} ─► mux16_8
АЦП-компаратор adc_cmp_h(ageb/aleb): данные АЦП vs порог cmp[13:0] (детект уровня/триггер)
```

- **Команда 2 (старт)**: установить флаг «идёт измерение» → разрешить `wrreq` всех FIFO;
  запустить проигрывание ЦАП; при `full` — авто-стоп; выставить `PB2`.
- **Команда 15**: `aclr` всех FIFO + сброс адреса ЦАП (по фронту).

## 7. Усиление (`gn`)

Усиление — **однобитный признак `gn`** (не плавный коэффициент). Хранится/отдаётся в
статусе и подмешивается как 2 старших бита в читаемые слова: `{gn,gn,ADC[13:0]}`,
`{gn,gn,DD[13:0]}`. Реальное масштабирование диапазона — на аналоговой части; `gn` — флаг
режима усиления, передаваемый вместе с данными.

## 8. Тактирование

`pll_16_100`: вход 16 МГц (пин G22) → `c0`=**64 МГц** (`clk64`, 4/1) и `c1`=**50 МГц**
(`clk50`, 25/8). 64 МГц — базовый такт ЦАП; `freq[1:0]` делит его для АЦП (64/32/16/8 МГц).

  5. `const_FFF`, пины `BRD[3:0]`, `PE2/PE6`, `TEST*` — назначение.

## Verilog-копия схемы (`Eddy_c.v`)

Создана **структурная копия 1:1** схемы `Eddy_c.bdf` в Verilog:

- **`Eddy_c.v`** — топ-уровень (entity `Eddy_c`), сгенерирован детерминированно из нетлиста
  схемы скриптом `tools/gen_verilog.py` (связность шин разрешается по имени, как в Quartus).
  157 инстансов, 48 портов. Примитивы (AND2/OR2/NOT/WIRE/GLOBAL) развёрнуты в `assign`,
  триггеры — в модуль `dff_prim`, дифф-буферы — примитив `ALT_OUTBUF_DIFF`.
- **`dff_prim.v`** — поведенческий эквивалент примитива Altera DFF (фронт + асинхронные
  активные-низкие `clrn`/`prn`, `clrn` приоритетнее).
- Листовые модули переиспользованы как есть (AHDL/VHDL/IP) — гарантия идентичности.
  Недостающие перенесены из `User_library`: `dc16, mux21, mux4_1, imp_former1, cmp8,
  constA5, lpm_counter0` (.tdf), `74165_a` (.vhd).
- `Eddy_c.qsf`: `BDF_FILE` заменён на `VERILOG_FILE Eddy_c.v` + `dff_prim.v` и список
  листовых файлов; **все пины сохранены**. Файл `Eddy_c.bdf` оставлен на диске для справки.

**Проверка (Quartus II 13.1):** Analysis & Synthesis — **0 ошибок**. Прямое сравнение
синтеза BDF vs Verilog:

| | Регистры | Pins | Биты памяти | PLL | LE |
|--|--|--|--|--|--|
| Оригинал BDF | 388 | 129 | 1 146 880 | 1 | 724 |
| Verilog-копия | 388 | 129 | 1 146 880 | 1 | 725 |

Регистры/выводы/память/PLL — точное совпадение; +1 LE — артефакт маппинга `dff_prim`.
Предупреждения (clock-mux на `mode[7]`/`cmd3`, конверсия prn+clrn-регистров `v0..v3` в
защёлки) идентичны поведению исходных примитивов.

**Регенерация:** `python tools/gen_verilog.py` (перечитывает `Eddy_c.bdf` → пишет `Eddy_c.v`).

### Новая функция: непрерывный режим (ping-pong) + двухпортовая память — отклонение от BDF

Чтобы получать сигнал с АЦП **без пробелов**, нужна одновременная запись (АЦП) и чтение
(SPI). Старая `scfifo` (одноклоковая, такт мультиплексировался) так не умеет. Поэтому 4
канальных буфера заменены на модуль **`adc_chan`** (`adc_chan.v`) — **двухпортовая RAM
16384×14** (инферится в M9K): запись в домене такта АЦП, чтение в домене clk64. Один модуль
обслуживает оба режима.

- **Одиночный режим** (`mode[14]=0`): пишет `nsamp` отсчётов и стоп в обоих режимах.
  В режиме «4 точки» меняется частота стробов записи, но команда 10 по-прежнему
  задаёт число 14-битных слов ОЗУ, которое затем читает СТМ командой 3.
- **Непрерывный** (`mode[14]=1`): два банка по `cap_total/2`. Запись **никогда не
  останавливается** (кольцо, перезапись непрочитанного = без пробелов). При заполнении банка
  флаг банка переключается; СТМ читает завершённый банк, пока пишется другой.
- **Тест-режим** (`mode[15]=1`): канал 0 пишет значение ЦАП (`C1DD`={gn,gn,DD}) вместо C0D.
- **PB2 и PB6** = `ready0`: одиночный — «готов» (набрано N); непрерывный — номер пишущегося
  банка. **Статус (cmd4):** `[7]`=банк(ch0), `[3:0]`=ready/банк по каналам.
- `vX` теперь SR-защёлка (set=cmd2, reset=cmd15); стопом/заворотом управляет `adc_chan`.
- Чтение указателя: `rden = read_dX & cmd3` (только команда 3, канал — `control[1:0]`).
- Банки симметричны: непрерывный режим заворачивается ровно на `2*hsize`.

**Проверка:**
- Модуль `adc_chan` — unit-симуляция ModelSim (`sim/tb_adc_chan.v`): одиночный fill/stop +
  непрерывный ping-pong — все тесты OK.
- **Полночиповая gate-level симуляция** (`sim/tb_eddy_top.v`, netlist `Eddy_c.vo`, библиотеки
  Altera; запуск `sim/run_system_sim.sh`): эмулирует реальный протокол STM из `AlteraMain.c`
  (команда=PD[3:0]+строб PD7, SPI mode0 16-бит, CS пословно). Сценарии: одиночный захват+
  чтение (статус ident 0xA5, 8 отсчётов читаются последовательно), непрерывный режим (бит
  банка статуса переключается). **Все тесты PASSED.** Подтверждена сквозная работа:
  PD7→команда → cmd9/cmd10 запись → cmd2 захват → cmd3 чтение → MISO(74165+предзагрузка PA4)
  → SPI; статусное слово, банки.
- Независимый агент-аудит интеграции — корректных багов не найдено.
- Quartus: синтез+fit **0 ошибок**, память в M9K (1 146 880 бит). Остаток старого
  scfifo-пути (`FULL/EMPTY/usedw`, adc_clk-муксы) — мёртвая логика, оптимизируется.

### Новая функция: подтверждение команды стробом PD7 (W13) — отклонение от BDF

Раньше `cmd[3:0] = PD[3:0]` читалось **непрерывно** (внутренним клоком) — команда могла
ложно срабатывать, пока биты `PD` переключаются. Теперь STM пишет 4 бита команды атомарно
(BSRR) и **даёт импульс на PD7** (пин **W13**), а FPGA **защёлкивает `cmd[3:0]` по
нарастающему фронту PD7** (с двойной синхронизацией к `clk`). Команда удерживается до
следующего подтверждения — поэтому держится всю SPI-транзакцию (чтение/запись данных).
- Действия (cmd2 старт и т.п.) срабатывают по смене подтверждённой команды.
- Канал (`PD[5:4]`) и частота (`PD[9:8]`) подтверждением НЕ защёлкиваются (читаются
  непрерывно, как и раньше).
- Нюанс: две подряд **одинаковые** командой-действием не перезапустятся (нужна другая
  команда между ними — в реальном цикле перед cmd2 всегда идёт cmd15, так что работает).

### Фикс «первого отсчёта» ЦАП + проверка 4-точечного непрерывного режима

**Первый отсчёт ЦАП:** попытка фикса `DAC_RAM.wren = cmd1 & SS` оказалась **ненадёжной**
(в части таймингов ломала запись ЦАП) — **откатил** к `wren = cmd1` (исходное поведение,
артефакт первого отсчёта остаётся). Надёжный фикс — на стороне STM: в `Altera_ReadDACRAM`
убрать шаг `cmd1` (читать `cmd0→cmd13`; cmd13 сам сбрасывает адрес) — это давало чистое
чтение включая первый отсчёт.
ВАЖНО (требует проверки на плате): запись ЦАП продвигает адрес только если предел `cmp`
(команда 14) ≥ числа записываемых отсчётов; иначе `compar_addr` заворачивает адрес и форма
портится. Убедиться, что STM задаёт cmd14 ДО записи формы.

**4-точечный непрерывный режим (`sim/tb_4point.v`):** проигрывание ЦАП (cmd6) пересекает
порог компаратора → строб `out_ad1` → захват АЦП; банк переключается (sim подтвердил).
Частота отсчётов = `N_точек × (f_DAC / длина_формы)`, f_DAC≈64 МГц. Для полной формы
(16384): ≈4×3.9кГц ≈ **15.6 кГц** — намного ниже скорости чтения STM (SPI 3.375 МГц / 16 ≈
**211 квыб/с**), запас ~13×. Для коротких форм (≤~1024) частота 4 точек может превысить
211 кГц → тогда поднять SPI или удлинить форму.

### Исчерпывающая проверка команд (gate-level sim) — найденные и исправленные проблемы

Полный тест каждой команды/регистра (`sim/tb_eddy_full.v`, gate-level `.vo`, протокол STM
из `AlteraMain.c`). Найдено и **исправлено 2 реальные проблемы**:

1. **Чтение регистров cmd11/cmd12/cmd14 возвращало не тот регистр.** STM читает delay/порог/
   reset-addr командами их записи (cmd11/12/14), но `mx_ctrl` маппил на чтение только
   cmd0/3/4/5/8/10/13 → возвращался `mode`. **Фикс:** расширил `mx_ctrl.tdf` (cmd11→delay,
   cmd14→reset-addr, cmd12→threshold) и завёл порог `cmpr` на свободный вход `data7x`
   (`mux_ctrl=111`).
2. **Чтение write+read регистров (cmd10/11/12/14) ЗАТИРАЛО регистр.** При чтении STM шлёт
   dummy `0xFFFF`, а т.к. эти команды в `cmd_w`, строб записи срабатывал и писал `0xFFFF`
   в регистр (критично для nsamp и порога — нет альтернативного кода чтения). **Фикс:**
   `not_rd = (dsp_d != 0xFFFF)` гейтит запись `wr10/wr11/wr12/wr14` — sentinel-слово при
   чтении регистр не портит (`0xFFFF` вне диапазона nsamp/порога/reset-addr; крайний случай —
   запись delay=0xFFFF блокируется).

**Результат теста (после фиксов) — всё PASS:** mode(cmd9/cmd0), nsamp(cmd10 w/r без порчи),
delay(cmd11 w / cmd8,cmd11 r), threshold(cmd12 w/r без порчи), reset-addr(cmd14 w / cmd5,cmd14 r),
статус(ident 0xA5, freq), d_run(cmd6/cmd7), ЦАП ОЗУ(cmd1 write / cmd13 read), многоканальный
захват (ch0=0..7, ch1=+100). Quartus синтез+fit **0 ошибок**.
Не покрыто тестом (драйв сложен): 4-точечный режим (`out_ad1`), поэлементное чтение банков
непрерывного режима.

### Новая функция: ограничение числа отсчётов АЦП (команда 10) — отклонение от BDF

**Команда 10 переопределена**: теперь это **запись/чтение регистра размера выборки**
`nsamp` (0…16384). Раньше cmd10 читала порог `cmpr` — теперь читает/пишет `nsamp`
(порог `cmpr` по-прежнему пишется командой 12, но обратно не читается).

Логика (в `Eddy_c.v`, блок «NEW FEATURE»):
- `wr10 = cmd10 & write_d` → `nsamp <= dsp_d[14:0]` (значение приходит по SPI, защёлкивается
  в конце транзакции). cmd10 добавлена в `cmd_w` (6-й вход OR6 `gn`→`cmd10`).
- Эффективный лимит: `eff_count = nsamp` в обоих режимах. В режиме «4 точки»
  каждый выбранный период даёт четыре слова: `delay0` и `delay1` после каждого
  из двух переходов через порог.
- Захват каждого канала останавливается по `usedw[k] >= eff_count` (гейт `wrreq=vX&~stopX`).
  → при полной форме и при 4 точках пишется/читается `nsamp` отсчётов.
  (`nsamp=0` или `=16384` ⇒ лимита нет, полный объём 16384.)
- Подключены `usedw1..3` всех каналов (раньше висели).

**Статусное слово (команда 4)** — `data6x` обновлено:
`{0xA5, bank=0, freq[1:0], d_run, bankfull3..0}`, где `bankfull[k] = FULL_k | stop_k`
(«банк заполнен» = достигнут лимит `nsamp` ИЛИ аппаратные 16384). Бит[7] = номер
банка ОЗУ B_S (пока всегда 0).

**Флаг канала 0** выведен на новый выход **PB6** (`PB6 = bankfull0`). PB4 в этом дизайне =
SPI MISO (занят), поэтому флаг разведён на отдельный пин PB6. `PB2` оставлен в исходном виде
(`PB2 = FULL0`, флаг окончания по команде 2 / аппаратный full@16384).
⚠️ **PB6 нужно назначить на физический пин FPGA** в `Eddy_c.qsf` (TODO рядом с PB4) — указать
вывод FPGA, разведённый под STM32-PB6 на плате.

**Проверено: синтез 0 ошибок**, +14 регистров (nsamp), LE 725→830.

### Исправление SPI на высокой частоте (отклонение от BDF)

**Симптом:** при максимальной SCK на STM приём по SPI сбоит (ловит данные, в следующем — 0);
надёжно только при SCK ~10–20 МГц.

**Причина:** передающий тракт MISO (два `74165`) загружался словом ответа `O_DAT` сигналом
`STLD = write_d | read_dd` — импульсом ~1 такт `clk64` (~15 нс), появляющимся **через** ~1
такт `clk64` **после** активации CS (`PA4↓`). На высокой SCK первый фронт SCK приходит
**раньше** загрузки → 74165 ещё держит старые данные (нули) → STM читает 0. На низкой SCK
загрузка успевает до первого фронта → работает.

**Фикс:** предзагрузка MISO **уровнем в простое**: `STLD = PA4` (CS снят → `PA4=1` →
74165 непрерывно держит `O_DAT` на выходе). `O_DAT[15]` валиден **до первого фронта SCK**
на любой частоте. Применено в `Eddy_c.v` (оба `74165`, было `.STLD(STLD)`) и в генераторе
`tools/gen_verilog.py` (сохранится при регенерации). Приём (MOSI, `lpm_shiftreg16`,
тактируется напрямую SCK) не менялся. **Проверено: синтез 0 ошибок.**

Дальнейшее повышение предела (если нужно >~40–50 МГц): SCK на clock-capable пин,
SDC-констрейнты на SCK/MISO/MOSI, либо замена MISO-тракта на чистый сдвиговый регистр по
`negedge SCK` (убирает задержку фабричного инвертора `SCK_I=~SCK`).

### A.6 Важный нюанс: `gn` = `GND`
`gn` приведён к `GND` (inst1). Поэтому: бит усиления в читаемых словах всегда 0
(`{0,0,data14}`); DFF с `D=gn` (v0..v3, d_run, a_en) работают как асинхронные защёлки
(set/reset через PRN/CLRN). При переписывании «усиление» можно реализовать как параметр,
но в текущем железе оно не используется.
## 2026-06-22 — AD9744/AD9649 continuous-mode correction

Primary datasheet requirements were applied to the converter interfaces:

- AD9744 data is registered in the output IOE and changes one half-cycle before
  the DAC rising clock edge (2.0 ns setup / 1.5 ns hold requirement).
- AD9649 data is captured by DCO-clocked IOE registers, pipelined once, then
  committed to M9K on the falling DCO edge. The ADC pipeline itself is eight
  conversion clocks and is a fixed physical-channel latency.
- `cmd6` now keeps both the DAC address counter and playback path clocked after
  STM selects `cmd3` or another command (`dac_play = cmd2 | d_run`).
- Playback uses `dac_play_ram.v`, an independent dual-clock shadow RAM. SPI
  readback remains on the legacy DAC RAM; command changes cannot stop or
  rephase the waveform sent to AD9744.
- Continuous ADC reads snapshot the completed bank when `cmd3` starts. Writer
  bank flips no longer move the SPI read pointer mid-transfer.
- Full-form test mode samples the registered digital DAC value from the
  forwarded ADC clock, avoiding an unnecessary AD9649 DCO crossing.

Verification:

- `tb_adc_chan`: PASS, including writer wrap during a snapshotted bank read.
- `tb_dac_play_ram`: PASS.
- Quartus map/fit/assembler: PASS, 0 errors.
- TimeQuest at slow 85 C: all declared clocks/interfaces PASS. Worst setup is
  +1.004 ns; worst hold is +0.409 ns. SPI is constrained to 27 MHz.
- Global sign-off remains WARN because the legacy AHDL/BDF-derived event-clock
  logic still leaves 12 inferred clocks and unrelated board I/O paths
  unconstrained.
