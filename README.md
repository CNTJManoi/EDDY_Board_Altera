# Eddy_c — FPGA система сбора и генерации сигналов

Проект реализует цифровую систему сбора аналоговых сигналов и генерации ЦАП на базе ПЛИС Altera Cyclone IV E.  
Управление осуществляется микроконтроллером STM32F7 по протоколу SPI.

---

## Аппаратная платформа

| Параметр | Значение |
|----------|----------|
| ПЛИС | Altera Cyclone IV E — EP4CE115F23C8 (484-pin FBGA) |
| Инструментарий | Altera Quartus II 11.0 / 13.1 |
| Язык RTL | Verilog (новые модули), AHDL / VHDL (legacy) |
| Системная тактовая | 100 МГц (PLL: 16 МГц → 100 МГц) |

---

## Архитектура системы

```
STM32F7  ──SPI──►  FPGA Cyclone IV E
                        │
                        ├──► ЦАП  (14-бит параллельный, LVDS тактирование)
                        │         DD[13..0],  DAC_CLKP / DAC_CLKN
                        │
                        ├──◄ АЦП0 (14-бит, C0D[13..0], тактирование C0DCO)
                        ├──◄ АЦП1 (14-бит, C1D[13..0], тактирование C1DCO)
                        ├──◄ АЦП2 (14-бит, C2D[13..0], тактирование C2DCO)
                        └──◄ АЦП3 (14-бит, C3D[13..0], тактирование C3DCO)
```

Усилители входных каналов управляются через отдельный маршрутизатор SPI → LTC2640.

---

## Структура модулей

```
Eddy_c.v                    ← топ-уровень (структурный Verilog, сгенерирован из BDF)
├── pll_16_100              ← Altera MegaWizard: 16→100 МГц PLL  (.qip)
├── ADC_RAM × 4             ← Altera MegaWizard: scfifo 16384×14 бит (.qip)
├── DAC_RAM                 ← Altera MegaWizard: scfifo 16384×14 бит (.qip)
├── adc_chan.v              ← буфер одного АЦП-канала (single-shot + ping-pong)
├── four_point_channel.v    ← четырёхточечный захват для одного канала
├── four_point_ctrl.v       ← контроллер триггера четырёхточечного захвата
├── amp_spi_router.v        ← прозрачный маршрутизатор SPI → LTC2640 (усилители)
├── dac_play_ram.v          ← воспроизведение данных из DAC FIFO
├── dff_prim.v              ← примитив D-триггера
├── mx_ctrl.tdf             ← декодер команд (AHDL, legacy)
└── pci_rail.tdf            ← старый PCI-интерфейс (AHDL, заменён SPI)
```

## Описание ключевых модулей

### `adc_chan.v` — буфер захвата АЦП-канала
Двухпортовая RAM 16384×14 бит, реализованная на M9K блоках.  
- **single-shot**: захват N отсчётов, затем останов до команды сброса
- **ping-pong** (непрерывный): две половины буфера, запись без пробелов  
- Пересечение тактовых доменов: wclk (DCO АЦП) → rclk (64 МГц) через двухтактный синхронизатор

### `four_point_channel.v` — четырёхточечный захват
Захватывает 4 отсчёта на период сигнала вокруг двух пороговых переходов:
- **point0, point1** — на `delay0`, `delay1` отсчётов после фронта (АЦП ≥ порог)
- **point2, point3** — на `delay0`, `delay1` отсчётов после среза (АЦП ≤ порог)

Режимы разрежения (`mode_div`): каждый 1 / 4 / 8 / 16 период сигнала.

### `four_point_ctrl.v` — контроллер триггера
Детерминированный порядок захвата: первая группа всегда стартует на восходящем переходе, что исключает сдвиг фазы при накоплении.

### `amp_spi_router.v` — маршрутизатор SPI → усилители
Прозрачно пробрасывает SPI4 (PE2/PE6/PE4) от STM32 к одному из четырёх LTC2640.  
Выбор канала: PD[5:4].

## Интерфейс SPI (STM32 ↔ FPGA)

| Параметр | Значение |
|----------|----------|
| Режим | SPI Slave, CPOL=0, CPHA=0 |
| Разрядность | 16 бит (MSB first) |
| Макс. SCLK | 25 МГц |
| Сигналы | SPI_SCK, SPI_MOSI, SPI_MISO, SPI_CSn |

### Формат кадра команды (STM32 → FPGA)
```
Биты [15:12]  CMD[3:0]   — код команды
Биты [11:0]   ARG[11:0]  — аргумент / данные
```

### Таблица команд

| CMD | Имя | Описание |
|-----|-----|----------|
| 0x0 | RESET | Программный сброс всех FIFO и состояний |
| 0x1 | START_DAC | Запуск воспроизведения ЦАП из FIFO |
| 0x2 | STOP_DAC | Останов ЦАП |
| 0x3 | WRITE_DAC | Запись отсчёта в FIFO ЦАП |
| 0x4 | START_ADC | Запуск захвата АЦП (выбор канала ARG[1:0]) |
| 0x5 | STOP_ADC | Останов АЦП |
| 0x6 | READ_ADC | Чтение отсчёта из FIFO АЦП |
| 0x7 | READ_ALL_ADC | Burst-чтение всех 4-х каналов |
| 0x8 | STATUS | Чтение статусного слова |

### Формат ответного кадра (FPGA → STM32)
```
Бит  [15]      ERR        — 1 = ошибка / FIFO пуст
Бит  [14]      BUSY       — 1 = выполняется операция
Биты [13:0]    DATA[13:0] — отсчёт или статус
```

## Назначение выводов (ключевые сигналы)

| Сигнал | Пин | Назначение |
|--------|-----|------------|
| `16MHz` | G22 | Входная тактовая |
| `DAC_CLKP` | R21 | LVDS дифференциальный такт ЦАП (+) |
| `DAC_CLKN` | R22 | LVDS дифференциальный такт ЦАП (−) |
| `DD[13:0]` | H22..P21 | Шина данных ЦАП 14 бит |
| `C0DCO..C3DCO` | B11,A11,B12,A12 | Тактирование АЦП 0..3 |
| `TEST0..TEST3` | F21,F22,E21,E22 | Отладочные выходы |

Полные назначения — в файле `Eddy_c.qsf`.

## Инструментарий и сборка

1. Открыть `Eddy_c.qpf` в **Altera Quartus II 11.0 / 13.1**
2. Запустить полную компиляцию: **Processing → Start Compilation**
3. Программирование ПЛИС: **Tools → Programmer**, файл `Eddy_c.sof`
4. Симуляция: ModelSim ASE, файлы `.vo` в `simulation/modelsim/`

---

## Системные требования

- Altera Quartus II 11.0 или 13.1 (рекомендуется 13.1)
- ModelSim-Altera ASE 13.1 (для симуляции)
- Windows 7/10/11
